<PageHeader content="在 Godot 中，内置函数 _physics_process() 和 _process() 有什么区别？" />

> `_physics_process()` 会在 `_process()` 之前被执行，并且 `delta 物理时间步长`是固定的。

::: tip

在 Godot 中，`_process()` 和 `_physics_process()` 是两种不同的内置函数，它们用于不同的目的和场景。

#### 1. **\_process()**:

- 这是所有继承自`Node`的类中都可以使用的标准函数。
- 它用于更新节点的状态，比如处理非物理相关的游戏逻辑、动画、用户输入等。
- `_process()`函数会在每帧被调用，但它的执行频率可以通过`process`参数（以秒为单位）来控制。如果设置为 0，它将只在逻辑帧（当窗口获得焦点时）被调用。
- `_process()`通常用于那些不依赖于物理计算的游戏逻辑，比如玩家控制、UI 更新、游戏状态管理等。

示例代码：

```gd
func _process(delta):
    # 这里是每帧都会执行的代码
    # 'delta' 是自上一帧以来经过的时间（秒）
```

#### 2. **\_physics_process()**:

- 这个函数专门用于处理物理相关的逻辑。
- 它同样在每帧被调用，但是它的调用顺序在`_process()`之前，这意味着物理模拟会在非物理逻辑之前更新。
- `_physics_process()`的`delta`参数是固定的，它表示自上一帧以来经过的时间，用于确保物理模拟的一致性。
- `_physics_process()`通常用于那些需要精确时间步长的应用，比如刚体的运动更新、碰撞检测和响应等。

示例代码：

```gd
func _physics_process(delta):
    # 这里是每帧物理更新时都会执行的代码
    # 'delta' 是固定的，表示物理时间步长
```

总结来说，`_process()`是一个通用的更新函数，用于处理游戏逻辑，而`_physics_process()`是专门为物理模拟和相关逻辑设计的更新函数。在设计游戏时，应该根据功能的不同选择合适的函数来实现代码逻辑，以确保游戏的性能和物理行为的正确性。

:::

::: danger 什么是物理时间步长？

在物理模拟中，"物理时间步长"（通常简称为"时间步长"或"物理步长"）是指用于更新物理状态的固定或恒定的时间量。这个概念在物理引擎中非常重要，因为它确保了物理模拟的一致性和可预测性，尤其是在处理碰撞、刚体运动和其他物理交互时。

物理时间步长通常由以下几个方面决定：

1. **固定步长**：在某些物理引擎中，物理时间步长是固定的，这意味着无论游戏的运行速度如何，物理模拟都会以相同的时间间隔更新。这有助于避免累积误差和确保物理模拟的稳定性，特别是在高速运动或精确碰撞检测中。

2. **动态步长**：在其他情况下，物理时间步长可能会根据系统的性能动态调整。如果硬件资源充足，物理引擎可能会使用较小的时间步长来提高模拟的精确度；而在资源受限的情况下，可能会增加时间步长以保持流畅的游戏体验。

3. **精度与性能的平衡**：选择物理时间步长时需要在模拟的精度和性能之间找到平衡。较小的时间步长可以提供更精确的物理模拟，但可能会增加计算负担；较大的时间步长可以提高性能，但可能会导致物理模拟不够精确，出现不稳定或不真实的行为。

在 Godot 中，`_physics_process()`函数使用的时间步长是固定的，这是为了确保物理模拟的稳定性和一致性。开发者可以通过设置`PhysicsServer`的`set_fixed_time_step()`函数来调整这个时间步长，以适应不同的物理模拟需求。通常，这个时间步长设置为 1/60 秒（约 16.67 毫秒）或 1/120 秒（约 8.33 毫秒）是比较常见的，这些值分别对应于 60Hz 和 120Hz 的物理更新频率。

:::
